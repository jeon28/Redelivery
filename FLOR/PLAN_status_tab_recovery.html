3. Stage 2 — VOID 캔슬 (체크박스 연동)

3.1 V1 VOID 트리거 분석 (라이브 캡처)

Stage 1 안정화 후 PPR 행이 잡힌 상태에서 다음을 확인:

  1. + More 확장 패널 내부 또는 행 옆 VOID/Cancel 버튼·메뉴 위치. 가능성:
    • 확장 패널 안 액션 영역 (예: "Cancel", "Void", "삭제" 라벨)
    • 행 오른쪽 끝 케밥(⋮) 메뉴 → "Cancel Redelivery"
    • 전용 컬럼 (예: "Actions") 의 아이콘 버튼
  2. 클릭 후 컴펌 다이얼로그 텍스트와 "Yes/No" 버튼 라벨
  3. VOID 후 해당 행 status 변화 — Open → VOID/Cancelled 로 바뀌는지, 사라지는지
  4. 여러 컨테이너가 한 PPR 아래 묶인 경우 — 단건 VOID vs 전체 VOID 동작 차이
  5. 권한 / 정책상 VOID 불가 케이스 (예: 시한 경과, Closed 상태) 의 에러 메시지

분석 결과는 ANALYSIS.md 의 "취소(Cancel) 흐름" 섹션에 기록 (TEXA ANALYSIS 처럼).

안전 원칙: V1 분석 단계에선 VOID 버튼을 실제로 클릭하지 않는다. DOM 셀렉터 / 라벨 / 컴펌 텍스트만 캡처. 첫 실 클릭은 V3 검증 단계에서 사용자 명시 승인 + 1건 한정 테스트.

async def cancel(self, items, region):
    # 입력 순서 보존을 위한 order 리스트 + outcomes 매핑
    order = [(it["container_no"], it["booking_ref"]) for it in items]
    outcomes: dict[tuple[str, str], tuple[bool, str | None]] = {}

    # FLOR 은 PPR 단위로 VOID — booking_ref 별 그룹핑
    groups: dict[str, list[str]] = {}
    for cn, bk in order:
        groups.setdefault(bk, []).append(cn)

    await self._navigate_to_status()

    for bk_ref, containers in groups.items():
        try:
            grp = await self._void_ppr(bk_ref, containers)
        except Exception as exc:
            logger.error("FLOR VOID %s 실패: %s", bk_ref, exc)
            grp = {cn: (False, f"오류: {exc}") for cn in containers}
        for cn in containers:
            outcomes[(cn, bk_ref)] = grp.get(cn, (False, "처리 결과 미확인"))

    return [
        {
            "container_no": cn,
            "booking_ref": bk,
            "cancelled": ok,
            "reason": None if ok else reason,
        }
        for (cn, bk), (ok, reason) in
        ((k, outcomes[k]) for k in order)
    ]

3.3 V3 안전장치

  • 사용자 컴펌은 프론트에서 이미 처리 (취소 모달 + "확인" 버튼). 백엔드는 컴펌된 요청만 받아서 실행. 추가 백엔드 컴펌 불필요.
  • 드라이런 모드 (선택): FLOR_VOID_DRYRUN=1 환경변수로 행 찾기/확장까지만 하고 VOID 클릭 직전 abort. 운영 첫 배포 직후 1~2일만 활성 후 끔.
  • idempotency: 이미 VOID/Cancelled 인 행 발견 시 cancelled=True, reason="이미 취소됨" 으로 처리 (중복 클릭 보호).
  • 다중 컨테이너 동일 PPR: PPR 단위로 한 번에 VOID 되므로 그룹 전체 동일 outcome. 프론트에서 일부만 체크해 요청 보내도 같은 PPR 의 나머지 컨테이너에 대한 결과는 "함께 취소됨" 으로 표시할지 정책 결정 필요 (V1 분석 결과 보고 확정).
  • 실패 분류: 시한 경과 / 권한 / 네트워크 / VOID 직후 검증 실패 — reason 텍스트 명확화.
