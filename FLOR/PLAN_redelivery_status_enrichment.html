FLOR 반납번호/유효기간 가시화 + 다건 입력 안내 계획

증상: 흥아라인 / FLOR / 부산 / KRPUS03 으로 DFSU7597484 조회 시 ✅ 가능 + depot 만 표시되고 반납번호 / 유효기간 / 불가사유 컬럼은 "-". FLOR 사이트의 "Unit Detail" 영역에는 Unit has been already precleared ( Redelivery number: PPR78148, Order Date: 2026-05-20, Depot Code: KRPUS03 ) 가 명확히 표시되며, 정확한 유효기간(Expiry Date 2026-07-19)은 Redelivery Status 탭의 + More 펼침에서만 노출됨.

1. 데이터 위치 정리

데이터위치비고
PPR 번호 (반납번호) Apply Redelivery Step 3 의 Unit Detail Info 셀 이미 발급된 케이스만 노출. 신규는 Confirm 클릭 후에야 발급됨 (Phase B, 본 PR 범위 외)
Order Date
(승인 일자, 유효기간 아님)		 Apply Step 3 Info 셀 / Redelivery Status 행 예: 2026-05-20
Expiry Date (유효기간) Redelivery Status 탭 → unit 검색 → + More 펼침 예: 2026-07-19. 사용자 화면 "유효기간" 컬럼에 표시할 값

2. 기존 코드 자산

4. 구체적 변경

A. _parse_step3 sweep fallback

기존 invalid/valid 테이블 분류 후, booking_ref가 비어 있는 컨테이너 한정으로 페이지 전체 텍스트에서 패턴 매칭: 

# 컨테이너 번호 뒤 300자 이내에서 "Redelivery number: PPR####" 추출
sweep = await self._sweep_precleared(list(results.keys()))
for unit, info in sweep.items():
    if results[unit].get("booking_ref"):
        continue
    results[unit].update({
        "available": True,
        "status": "available",
        "booking_ref": info["ppr"],
        "reason": None,
        # close_date 는 Status 탭 enrichment 에서 Expiry Date 로 채움
    })

신규 _sweep_precleared(containers): document.body.innerText 를 JS 로 가져와 컨테이너별 정규식 검색.

B. _navigate_to_status()

Apply 탭과 같은 패턴으로 "Redelivery Status" 텍스트 클릭. 탭 없으면 REDELIV_URL로 재진입 후 재시도. 

async def _navigate_to_status(self) -> None:
    status_tab = self.page.locator("text=Redelivery Status").first
    if await status_tab.count() == 0:
        await self.page.goto(REDELIV_URL, wait_until="networkidle", timeout=30_000)
        await asyncio.sleep(2.0)
        status_tab = self.page.locator("text=Redelivery Status").first
    await status_tab.click(timeout=5_000)
    try:
        await self.page.wait_for_load_state("networkidle", timeout=10_000)
    except Exception:
        pass
    await asyncio.sleep(1.5)

C. query() 끝에 enrichment 호출

# 8. Step 3 파싱 (현재)
await self._parse_step3(results, depot_str)

# 9. NEW: PPR 있는 컨테이너 대상 Status 탭 enrichment
to_enrich = [c for c, r in results.items() if r.get("booking_ref")]
if to_enrich:
    try:
        await self._navigate_to_status()
        for c in to_enrich:
            try:
                row = await self._search_one(c)
                if row.get("close_date"):  row["close_date"]  # ← Expiry Date
                if row.get("depot"):       results[c]["depot"]       = row["depot"]
                if row.get("over_caps") is not None:
                    results[c]["over_caps"] = row["over_caps"]
                if row.get("close_date"):
                    results[c]["close_date"] = row["close_date"]
                if row.get("booking_ref"):
                    results[c]["booking_ref"] = row["booking_ref"]
            except Exception as exc:
                logger.warning("FLOR %s enrich 실패: %s", c, exc)
    except Exception as exc:
        logger.warning("FLOR Status 탭 진입 실패 (enrichment 스킵): %s", exc)

실패해도 결과 행은 유지 (가능 + booking_ref 만 있는 상태로 표시).

D. Frontend 다건 입력 안내

SearchForm.tsx 컨테이너 textarea 아래, 기존 isWip 안내문과 같은 노란색 박스 스타일:

⚠️ FLOR 6개 이상 입력 — 이미 반납번호가 발급된 컨테이너가 있을 경우 1건당 ~6초 추가 소요됩니다.

표시 조건:

  • templateKey(lessor) === 'FLOR'
  • containers.length > 5

입력 차단은 하지 않음. 조회 버튼은 그대로 활성.
