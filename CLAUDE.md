# 프로젝트 규칙

## 협업 규칙

- **코딩 시작 전 반드시 협의**: 무엇을 만들지, 어떻게 만들지 명확히 합의가 된 후에만 구현을 시작한다.
- 방향이 불분명하거나 합의되지 않은 사항은 구현 전에 먼저 질문한다.
- 합의된 내용을 변경해야 할 경우 먼저 알리고 승인을 받는다.
- **개발 전 계획 필수**: 코딩 시작 전에 반드시 개발 계획을 `.md` 파일로 작성하고 제출한다.
- 계획 검토 및 협의가 완료된 후에만 코딩을 시작한다.
- 계획 내용: 목표, 기술 스택, 폴더 구조, 주요 기능, 작업 순서 등을 포함한다.

## 프로젝트 구조 규칙

- **임대사별 독립 개발**: 각 임대사(리스사)는 웹 접근 방식이 다르므로 임대사별로 개별 프로그램을 제작한다.
- **임대사별 폴더 분리**: 임대사마다 독립된 폴더로 관리한다.
- **순차 개발**: 한 임대사씩 차근차근 개발하며, 한 임대사가 완성된 후 다음 임대사로 진행한다.
- 각 임대사 폴더는 해당 임대사의 웹 접근 방식, 로직, 설정을 독립적으로 포함한다.
