# 장금상선 개발 규칙

> 본 폴더는 장금상선(SINOKOR) 전용 작업 공간입니다.
> 루트 `../CLAUDE.md` 규칙을 먼저 따르고, 본 문서의 규칙을 추가로 적용합니다.

## 1. 선사 식별 정보

| 항목 | 값 |
|------|-----|
| 선사명 | 장금상선 (SINOKOR) |
| 코드 prefix | `SK` |
| 환경변수 형식 | `SK_<LESSOR>_ID`, `SK_<LESSOR>_PW` |

## 2. 사용 임대사(리스사) 목록

| 임대사 | 코드 | 접근 방식 | 개발 순위 |
|--------|------|----------|----------|
| Textainer | TEXA | 웹 자동화 (Playwright) | 1순위 |
| Triton | TRIT | 웹 자동화 (Playwright) | 2순위 |
| Touax | GOLD / GLOD | 웹 자동화 (Playwright) | 3순위 |
| Florens | FLOR | 웹 자동화 (Playwright) | 4순위 |
| SeaCo | GESE | 웹 미작동 — 보류 | 5순위 |
| BCON, CARL, CAIC, GCIC, BLUE | - | 이메일 발송 | Phase 3 |

## 3. 문서/코드 위치 매핑

- **분석 문서**: 본 폴더 안에 `<LESSOR_CODE>_ANALYSIS.md` 형식으로 저장
  - 예: `장금상선/TEXA_ANALYSIS.md`, `장금상선/TRIT_ANALYSIS.md`
- **스크래퍼 구현**: `../scraper/scrapers/<lessor_code>.py`
  - 예: `scraper/scrapers/texa.py`, `scraper/scrapers/trit.py`
- 본 폴더에는 분석/규칙 등 문서만 두고, 실행 코드는 두지 않는다.

## 4. 작업 규칙

- **분석 → 합의 → 구현 → 테스트** 순으로 진행하며, 각 단계 사이에 사용자 승인을 받는다.
- 임대사 사이트 분석은 라이브 시연 또는 사용자 제공 자료를 바탕으로 수행한다.
- 동일 임대사 코드라도 흥아라인(HA)과 장금상선(SK)은 별도 계정/주의사항을 가지므로 폴더를 절대 혼용하지 않는다.
- 환경변수에 실제 자격증명을 커밋하지 않는다. `scraper/.env`만 사용한다.

## 5. 주의사항

- **중복 예약 금지**: 임대사 사이트의 예약/Book 버튼은 실 발급을 의미하므로, 동일 컨테이너 중복 실행을 방지하는 로직을 항상 둔다.
- **사이트 변경 대응 순서**: 사이트 UI/플로우가 변경되면 먼저 분석 문서를 갱신한 후 스크래퍼 코드를 수정한다.
- **계정 보호**: 본 문서나 분석 문서에 실제 ID/PW를 기재하지 않는다. 키 이름만 표기한다 (예: `SK_TRIT_ID`).
