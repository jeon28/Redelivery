# GCIC 개발 규칙

> 임대사: GCIC (Global Container International Corp 추정)
> 코드: `GCIC`
> 본 임대사는 **장금상선 단독** 사용 (흥아라인 사용 안 함)
> 루트 `../CLAUDE.md` 규칙을 먼저 따르고, 본 문서의 규칙을 추가로 적용합니다.

## 1. 기본 정보

| 항목 | 값 |
|------|-----|
| 임대사명 | GCIC |
| 코드 | `GCIC` |
| 도메인 | `gcx.intermodalportal.com` (메모리 기준) |
| 로그인 URL | _(분석 시 기록)_ |
| 접근 방식 | 웹 자동화 (Playwright) |

## 2. 자격증명 키 규칙

| 선사 | ID 키 | PW 키 |
|------|-------|-------|
| 장금상선 (SK) | `SK_GCIC_ID` | `SK_GCIC_PW` |
| 흥아라인 (HA) | (사용 안 함) | (사용 안 함) |

- 흥아라인은 GCIC를 사용하지 않음.
- 본 문서 및 `ANALYSIS.md`에 실제 ID/PW 절대 기재 금지.

## 3. 위치 매핑

| 항목 | 경로 |
|------|------|
| 사이트 분석 문서 | `./ANALYSIS.md` |
| 스크래퍼 구현 | `../scraper/scrapers/gcic.py` |
| 공통 베이스 클래스 | `../scraper/scrapers/base.py` |
| 자격증명 환경변수 | `../scraper/.env` |

## 4. 작업 규칙

- **분석 → 합의 → 구현 → 테스트** 순. 각 단계 사이 사용자 승인.
- `intermodalportal.com` 도메인은 BCON/CARL(메일 임대사)과 공유 — UI 패턴 유사 가능성, 분석 시 참고.
- 사이트 변경 시 `ANALYSIS.md` 먼저 갱신 후 `gcic.py` 수정.

## 5. 분석 시 최소 캡처 항목

- 로그인 URL/폼 (캡차/2FA 유무)
- 반납 메뉴 진입 경로
- 입력 폼 (지역/포트/컨테이너 번호/날짜 등)
- 결과 화면 (가능/불가, 컬럼명)
- 반납번호 발급 트리거 + 단계 수
- 예약 취소 경로

## 6. 주의사항

- 실 발급 위험성 사전 검증. 안전 검증 전까지 발급 버튼 클릭 금지.
- 중복 예약 방지 로직 필수.
- 흥아라인이 사용하지 않는 임대사이므로 `company` 분기에서 SK 전용 처리.
- 계정 보호 (실 ID/PW 기재 금지).
