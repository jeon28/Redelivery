## 4. 작업 규칙

- **분석 → 합의 → 구현 → 테스트** 순으로 진행, 단계 사이 사용자 승인.
- 사이트 분석은 사용자 라이브 시연 또는 분석 스크립트로 수행, 결과를 `ANALYSIS.md`에 정리.
- 사이트 UI/플로우 변경 시 **`ANALYSIS.md` 먼저 갱신** 후 `gold.py` 수정.
- 선사 차이점은 `ANALYSIS.md` 끝의 "선사 차이점" 섹션에 기록.

## 5. 분석 시 캡처 항목 (최소)

- 로그인 URL 및 폼 필드명 (캡차/2FA 유무)
- 반납 조회 메뉴 진입 경로 또는 직접 진입 URL
- 입력 필드 (지역/포트/컨테이너 번호/날짜 등) 및 위젯 종류
- 결과 화면 구조 (가능/불가 표시, 컬럼명)
- 반납번호 발급 트리거 및 발급 후 상세 화면 (단계 수)
- 예약 취소 경로 (향후 cleanup 자동화 대비)

## 6. 주의사항

- **실 발급/예약 위험**: 분석 단계에서 발급 버튼 클릭이 실제 예약을 생성하는지 사전 확인. 안전 검증 전까지 발급 버튼 클릭 금지.
- **중복 예약 방지**: 동일 컨테이너 중복 실행 방지 로직 필수.
- **계정 보호**: 본 문서나 분석 문서에 실제 ID/PW 기재 금지.
