TEXA 로그인 실패 가시화 계획

사용자 증상: 장금상선 / TEXA / INCHON / TEMU7542020 1건 조회 시 화면에 아무 메시지 없이 "조회 중..." → "🔍 조회하기"로 즉시 복귀. 원인 후보로 로그인 실패 가능성이 큼. 본 계획은 로그인 단계의 실패를 단계별로 식별·가시화하여 진짜 원인을 좁힌다.

1. 현재 로그인 흐름 분석

1.1 모든 실패가 한 덩어리에 묻히는 구조

scraper/scrapers/texa.py:139-177 의 _fresh_login 본체는 8개 단계를 하나의 try/except로 묶고 있다. 어느 단계에서 깨졌는지 사용자에게는 물론 로그에도 일관된 형태로 남지 않는다. 

# _fresh_login (texa.py:139-177)
try:
    await self.page.goto(...)                                   # ① 메인 페이지 진입
    await self.page.evaluate(...)                               # ② 로그인 모달 열기 (JS click)
    await self.page.wait_for_selector("select", ...)            # ③ select 표시 대기
    await self.page.select_option("select", value=...)          # ④ Leasing Customer 선택
    await self.page.get_by_label("Login Name").fill(...)        # ⑤ ID 입력
    await self.page.get_by_label("Password").fill(...)          # ⑥ PW 입력
    await self.page.get_by_role("button", name="Login").click() # ⑦ 제출
    await self.page.wait_for_function(                          # ⑧ redirect 대기 (30s)
        "window.location.href.includes('tex.textainer.com')",
        timeout=30_000,
    )
except Exception as exc:
    logger.error(...)
    return False  # ← 어느 단계인지 정보 없이 False

base.py:62-69 의 run() 은 login() 이 False를 반환하면 일률적으로 RuntimeError("TEXA 로그인 실패") 로 던진다. 사용자가 화면에서 보게 될 메시지(가시화 작업 완료 시)는 이 generic 한 줄.

1.2 추가 약점

#위치문제
W1 _fresh_login 진입 전 자격증명 빈 값 사전 체크 없음. 빈 ID/PW로 ⑧단계에서 30초 대기 후 실패 — 사유 안 보임
W2 login():108 세션 체크가 "tex.textainer.com" in url 만 확인 — tex.textainer.com/Login.aspx 같은 URL이면 만료인데도 "valid"로 오판 가능
W3 login():125 세션 만료 fallback과 첫 fresh login 실패의 구분이 로그에만 남고 사용자에게는 동일 메시지로 보임
W4 base.py:66 f"{self.lessor} 로그인 실패" 고정 메시지 — 스크래퍼가 알아낸 구체 사유를 전달할 경로 없음

2. 변경 계획

2.1 _fresh_login 단계별 catch + 구체 메시지

각 단계를 작은 try/except 로 감싸고, 실패 시 self._login_error: str 에 구체 사유를 저장한 후 return False.

단계실패 시 사용자에게 보일 메시지
① gotoTEXA 메인 페이지 진입 실패 (네트워크 또는 사이트 다운)
② 모달 JS clickTEXA 로그인 모달 열기 실패 (사이트 UI 변경 가능)
③ select 대기TEXA 로그인 폼 로드 실패
④ select_optionTEXA 사용자 유형 선택 실패
⑤⑥ fillTEXA 로그인 양식 입력 필드 누락
⑦ Login 클릭TEXA Login 버튼 클릭 실패
⑧ redirect timeoutTEXA 자격증명 거부됨 또는 사이트 응답 지연 (portal 진입 30초 timeout)

2.2 자격증명 사전 체크 (W1 대응)

_fresh_login 맨 처음:

if not self.cred.get("id") or not self.cred.get("pw"):
    self._login_error = f"TEXA 자격증명 미설정 ({self.company})"
    return False

→ 30초 대기 없이 즉시 실패. 비밀번호 관리 페이지로 안내 가능한 상태.

2.3 세션 만료 판정 강화 (W2 대응)

현재 login():108 의 URL 도메인 체크에 추가 조건:

  • URL이 Login.aspx / /login / signin 포함 시 만료로 판정 (강제 fresh login)
  • 또는 portal 진입 후에도 "Login Name" 라벨이 frame 어딘가에 보이면 만료

두 번째는 비용 큼. 1번 URL path 체크만 추가하는 게 비용 대비 효과 좋음.

2.4 base.py 메시지 패스스루 (W3/W4 대응)

base.py:62-69 run():

if not await self.login():
    msg = getattr(self, "_login_error", None) or f"{self.lessor} 로그인 실패"
    raise RuntimeError(msg)

한 줄 추가. _login_error 속성을 안 가진 다른 스크래퍼는 영향 없음 (getattr 폴백 → 기존 메시지).

2.5 (선택) 프론트 측 가시화 마무리

조금 전 frontend/app/api/query/route.ts의 status 코드 전달 1줄은 이미 적용됨. 남은 한 곳: 

// SearchForm.tsx::handleQuery
if (!res.ok || data.error || data.detail) {
  throw new Error(data.detail || data.error || `조회 실패 (HTTP ${res.status})`)
}

이 ~3줄을 추가하면 위 2.1~2.4에서 던진 메시지가 화면 빨간 박스에 그대로 표시됨.
